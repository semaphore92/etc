** 서비스 디스커버리 패턴

 - 디스커버리 서버는 가변적인 모드 서비스의 정보들을 고유ID에 매핑하여 관리
   1) 자신의 정보를 디스커버리 서버에 등록
   2) 다른 서비스들의 정보를 조회

  
1. 디스크버리의 구현을 쉽게 도와주는 것이 Spring Cloud의 Eureka
2. Ribbon <- L4의 로드밸런싱 역할을 해주는 라이브러리
3. 내부에서는 디스커버리 서버를 통해 이용하는데 문제 없지만,
   외부클라이언트에서는 각 서비스들의 정보를 어떻게 알아야하는가
   API Gateway <- 도와주는 도구로 Spring Cloud Zuul1, Zuul2, Spring Cloud gateway

4. Config Server <- 중앙집중식 환경설정
